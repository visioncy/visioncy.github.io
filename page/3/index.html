下载和安装rabbitmq

sudo wget https://github.com/rabbitmq/rabbitmq-server/releases/download/v3.8.5/rabbitmq-server-3.8.5-1.el7.noarch.rpm
sudo rpm -import https://github.com/rabbitmq/rabbitmq-server/releases/download/v3.8.5/rabbitmq-server-3.8.5-1.el7.noarch.rpm.asc

sudo yum install rabbitmq-server-3.8.5-1.el7.noarch.rpm

设置开启自动启动和启动

sudo chkconfig rabbitmq-server on
sudo systemctl start rabbitmq-server
sudo rabbitmqctl status

开启web管理界面

sudo rabbitmq-plugins enable rabbitmq_management

开放外部端口

5672和15672端口， 15672为web 管理端口

添加用户和外网访问

sudo rabbitmqctl add_user admin 123456
sudo rabbitmqctl set_user_tags admin administrator

sudo rabbitmqctl set_permissions -p "/" admin '.*' '.*' '.*'
sudo rabbitmqctl list_user_permissions admin

安装delay插件（默认目录）

sudo wget https://github.com/rabbitmq/rabbitmq-delayed-message-exchange/releases/download/v3.8.0/rabbitmq_delayed_message_exchange-3.8.0.ez

sudo cp rabbitmq_delayed_message_exchange-3.8.0.ez /usr/lib/rabbitmq/lib/rabbitmq_server-3.8.5/plugins/

sudo rabbitmq-plugins enable rabbitmq_delayed_message_exchange

附：rabbitmq常见命令，非root用户请使用sudo

添加用户

rabbitmqctl add_user

删除用户

rabbitmqctl delete_user

修改用户密码

rabbitmqctl change_password

清除用户密码（该用户将不能使用密码登陆，但是可以通过SASL登陆如果配置了SASL认证）

rabbitmqctl clear_password

设置用户tags（相当于角色，包含administrator，monitoring，policymaker，management）

rabbitmqctl set_user_tags

列出所有用户

rabbitmqctl list_users

创建一个vhosts

rabbitmqctl add_vhost

删除一个vhosts

rabbitmqctl delete_vhost

列出vhosts

rabbitmqctl list_vhosts [ …]

针对一个vhosts给用户赋予相关权限；

rabbitmqctl set_permissions [-p ]

清除一个用户对vhosts的权限；

rabbitmqctl clear_permissions [-p ]

列出哪些用户可以访问该vhosts；

rabbitmqctl list_permissions [-p ]

列出用户访问权限；

rabbitmqctl list_user_permissions

centOS 安装配置

1. 安装常用包

客户端相关操作
4.1、生成客户端证书库
keytool -validity 36500 -genkeypair -v -alias client -keyalg RSA -storetype PKCS12 -keystore client.p12 -dname “CN=clients.itjoyee.com,OU=jiajianfa,O=jiajianfa,L=Wuhan,ST=HuBei,c=cn” -storepass 123456 -keypass 123456

4.2、从客户端证书库中导出客户端证书
keytool -export -v -alias client -keystore client.p12 -storetype PKCS12 -storepass 123456 -rfc -file client.cer

注:客户端证书可以产生多个.

4.3、将客户端证书导入到服务器证书库(使得服务器信任客户端证书,服务器端用此验证客户端的合法性)
keytool -import -v -alias client -file client.cer -keystore server.keystore -storepass 123456

4.4、查看服务端证书中信任的客户端证书
keytool -list -keystore server.keystore -storepass 123456

centOS 安装 mongodb

1. 添加安装源

sudo vim /etc/yum.repos.d/mongodb-org-4.2.repo

[mongodb-org-4.2]
name=MongoDB Repository
baseurl=https://repo.mongodb.org/yum/redhat/$releasever/mongodb-org/4.2/x86_64/
gpgcheck=1
enabled=1
gpgkey=https://www.mongodb.org/static/pgp/server-4.2.asc

2.安装mongodb常用包

sudo yum install -y mongodb-org

移动数据库

#复制数据库
rsync -av /var/lib/mongo /data/database/mongodb/
#备份旧数据库
sudo mv /var/lib/mongo /var/lib/mongo.bak

如果运行数据库是提示没权限，可以进行授权

#Change owner and group
sudo chown -R mongod:mongod /data/mongodb/mongo
#Change context
sudo chcon -R -u system_u -t mongod_var_lib_t /data/mongodb/mongo/

3.修改数据库保存路径
vim /etc/mongod.conf

#启动安全验证
security:
    authorization: enabled

初步化数据库

添加超级管理员

mongo

use admin
db.createUser(
  {
    user: "xhadmin",
    pwd: "abc123",
    roles: [ { role: "userAdminAnyDatabase", db: "admin" }, "readWriteAnyDatabase" ]
  }
)

创建数据库用户

use test
db.createUser(
  {
    user: "myTester",
    pwd: "xyz123",
    roles: [ { role: "readWrite", db: "test" },
             { role: "read", db: "reporting" } ]
  }
)

集群

replication:
  oplogSizeMB: 20000
  replSetName: "flygramway"

config = {
   "_id":"flygramway",
  "members":[
    {"_id":0,"host":"10.148.0.51:27017", arbiterOnly:true},
    {"_id":1,"host":"10.148.0.52:27017"},

rs.status()

mongod增加keyFile

openssl rand -base64 756 > /data/key_file
chmod 400 /data/key_file

  authorization: enabled

openfire 优化

原来连接与登录共收发20条信息
修改后连接与登录共收发8条信息

1.[Server Settings]->[Registration & Login]->[SASL Mechanisms]只保留[SCRAM-SHA-1]（暂时测试不确认是否有优化）
并且xmpp.client.tls.policy改为:disabled

  • required 必需
  • optional 可选
  • disabled 关闭
  • legacyMode 一种要求连接立即加密的策略（与“必需”策略相反，它允许最初未加密的连接通过StartTLS加密。

这样可以减少一次接收信息
